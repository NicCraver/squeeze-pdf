# squeeze-pdf 项目介绍

## 概述

**squeeze-pdf** 是一款已实现的跨平台桌面应用（macOS / Windows），用于将 PDF 文件压缩到指定体积上限，同时尽可能保持可读清晰度。适合邮件附件、上传限制、归档存储等需要批量控制 PDF 体积的场景。

## 核心能力

- 图形界面拖拽或选择 PDF，批量压缩
- 在不超过目标体积（默认 **1 MB**）的前提下，自动选择最佳清晰度参数
- 通过页面栅格化 + JPEG 压缩，有效去除巨型嵌入字体等导致体积膨胀的内容
- 多文件并行处理，WebSocket 实时进度推送
- 跨平台桌面客户端（macOS `.app` / Windows `.exe`）

## 工作原理

1. 使用 [PyMuPDF](https://pymupdf.readthedocs.io/)（`fitz`）打开源 PDF
2. 将每一页按给定 **缩放倍率（zoom）** 渲染为位图，再以指定 **JPEG 质量** 编码
3. 将栅格化后的页面写入新 PDF
4. 在预设的 zoom / quality 组合中搜索：优先高清晰度，逐步降低直到输出文件 ≤ 目标体积
5. 若所有组合仍超过上限，则采用体积最小的版本并显示警告

### 参数搜索策略

| 类型 | 预设值 |
|------|--------|
| 缩放倍率 (zoom) | 2.0 → 1.75 → 1.5 → 1.25 → 1.0 → 0.85 → 0.72 → 0.6 |
| JPEG 质量 | 85 递减至 20（步长 5） |

搜索顺序为「先固定 zoom、遍历 quality」，再降低 zoom，从而在体积约束下尽量保留清晰度。

## 环境要求

- **Python** 3.12+
- **Node.js** 22.12+（前端开发/构建）
- **Vite+**（`vp` CLI）
- macOS：系统 WebKit（pywebview 内置）
- Windows：WebView2 Runtime

## 开发与构建

### 安装依赖

```bash
# Python
python -m venv .venv
source .venv/bin/activate
pip install -e ".[dev]"

# 前端
vp install
```

### 开发模式

```bash
# 构建前端到 backend/static/
vp run desktop-ui#build

# 启动桌面客户端（内嵌 FastAPI + pywebview 窗口）
python -m client.app
```

或使用快捷脚本：

```bash
./scripts/dev.sh
```

前端热更新开发（可选）：

```bash
vp run desktop-ui#dev   # 终端 1
python -m client.app    # 终端 2
```

### 生产构建

```bash
# macOS 一键构建（前端 + PyInstaller + DMG）
./scripts/build-macos.sh
# 产物: dist/squeeze-pdf.app, squeeze-pdf-macOS.dmg

# Windows 一键构建（需 Inno Setup 6）
./scripts/build-win.ps1
# 产物: squeeze-pdf-Windows-Setup.exe
```

### GitHub Actions 自动打包（推荐）

推送到 GitHub 后由 CI 在云端构建（参考 py-caj-pdf 项目）：

1. 创建 GitHub 仓库并 `git push` 到 `main` / `master`
2. 自动运行 **Build** 工作流：`pytest` → 双平台 PyInstaller → 发布 Release
3. 产物：
   - `squeeze-pdf-macOS.dmg`
   - `squeeze-pdf-Windows-Setup.exe`
4. push tag（如 `v0.2.0`）会创建对应版本 Release；push 分支会更新 `latest` 预发布

在 macOS 上拉取 Windows 安装包：

```bash
./scripts/fetch_windows_build.sh
```

也可在 **Actions** → **Build** → **Run workflow** 手动触发。

## UI 设计

- **布局**：左侧参数面板（体积上限、输出目录），右侧文件列表与进度
- **交互**：拖拽多文件、单文件失败不中断批次
- **视觉**：中性灰 + teal 强调色，支持浅色/深色模式
- **原则**：极简、操作路径 ≤ 3 步完成压缩

## 注意事项

- **栅格化会丢失矢量与可选文字层**：压缩后的 PDF 以图片形式呈现页面，无法再选中或复制原文文字（除非 OCR）。适合以「阅读/打印」为主的场景。
- **清晰度与体积的权衡**：目标体积越小，程序会采用更低的 zoom 或 JPEG 质量；复杂页面或扫描件可能难以压到极小体积。
- **无 CLI**：所有操作通过桌面 GUI 完成；旧版 `squeeze.py` 已移除。

## 适用场景

- 批量将 PDF 压到邮件或系统的附件大小限制内
- 统一归档体积，节省存储空间
- 对含超大嵌入字体、高分辨率图片的 PDF 做「瘦身」
